ARQUITECTURA HEXAGONAL - FLUJO SIMPLIFICADO
=============================================

Petición HTTP
    ↓
┌─────────────────────────────────────┐
│ INFRASTRUCTURE: ExpressController   │
│ - Valida query params               │
│ - Autenticación JWT (middleware)    │
│ - Convierte request → comando       │
└──────────────┬──────────────────────┘
               ↓
┌─────────────────────────────────────┐
│ APPLICATION: TareaService           │
│ - Orquesta comandos                 │
│ - Valida reglas de negocio (App)   │
│ - Llama adaptadores vía puertos     │
└──────────────┬──────────────────────┘
               ↓
┌─────────────────────────────────────┐
│ DOMAIN: Comandos + Entidades        │
│ - CrearTareaCommand.execute()       │
│ - Valida reglas de dominio          │
│ - Construye entidad pura            │
└──────────────┬──────────────────────┘
               ↓
┌─────────────────────────────────────┐
│ INFRASTRUCTURE: Adaptador SQLite    │
│ - TareaRepositoryImpl.save(tarea)   │
│ - Ejecuta INSERT en BD              │
│ - Maneja transacciones              │
└──────────────┬──────────────────────┘
               ↓
         SQLite Database


SEGURIDAD
=========

1. Autenticación (JWT):
   - Header: Authorization: Bearer <token>
   - Middleware authMiddleware verifica firma
   - Extrae usuario en req.user

2. Autorización (Roles):
   - Middleware checkRole(UserRole.ADMIN)
   - Protege rutas por rol
   - Roles: MIEMBRO, LIDER, ADMIN

3. Transacciones ACID:
   - BEGIN TRANSACTION
   - Operaciones múltiples
   - COMMIT o ROLLBACK automático


i18n (INTERNACIONALIZACIÓN)
============================

- Archivos JSON en shared/i18n/{es,en}.json
- i18nManager.ts con método t(lang, key)
- Uso: i18n.t('es', 'tarea.creada')
- Endpoint: GET /api/i18n/{es|en}


ENDPOINTS PRINCIPALES
======================

GET    /health                           → Health check
GET    /api/i18n/{es|en}                → Traducciones

GET    /api/proyectos                   → Listar proyectos
POST   /api/proyectos                   → Crear proyecto
GET    /api/proyectos/:id               → Obtener proyecto
PUT    /api/proyectos/:id/archive       → Archivar (ADMIN)

GET    /api/tareas                      → Listar tareas
POST   /api/tareas                      → Crear tarea
GET    /api/tareas/:id                  → Obtener tarea
PUT    /api/tareas/:id/move             → Cambiar estado
PUT    /api/tareas/:id/assign           → Asignar responsable
DELETE /api/tareas/:id                  → Eliminar tarea
GET    /api/tareas/proyecto/:id/dashboard → Stats

GET    /api-docs                        → Swagger UI
